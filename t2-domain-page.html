📋 T2 Builder Notes — How to Recreate This in SharePoint

  • Page type: Build this as a modern SharePoint page under the T1 master hub site. Treat it as a section page, not a separate site experience.
  • Purpose of T2: This is a navigation layer, not a content-heavy destination. Its primary job is routing users down to T3 category pages.
  • T3 Category Cards: Use the Quick Links web part (Tiles layout with descriptions enabled) — one tile per T3 category page. Or use a Card Layout section with individual Image + Text combinations per card.
  • Card count badges: These are manual (set in the Quick Links description field) or use a Highlighted Content web part count. They can also be driven from a metadata list.
  • Featured Policies strip: Use Highlighted Content web part — filter on a "Featured" metadata column = Yes, scoped to this page context.
  • Recent Updates list: Use News web part (List layout) or Document Library web part filtered by Modified date — showing newest T4 content pages first.
  • Section stats bar: Build with a Text web part custom table or a List web part count driven by your policy register metadata.
  • Page Activity: Add activity/updates web parts to show recent edits and publish events related to this section.
  • Navigation pattern: Keep this page clean and routing-first. Surface T3 category links both in cards and optional quick link lists.
